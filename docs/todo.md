# InteropHub ES / External Initiative Coordination Ideas

## Core Direction
- Emerging Standards tracks and coordinates ecosystem activity
- ES topics may be moved by outside organizations/projects
- ES acts as visibility + engagement + continuity layer
- Avoid governance ownership model
- Support IIP, IGEG, HL7, CDC, Connectathons, external pilots, etc.

- Examples:
  - IIP
  - IGEG

  - WHO
  - CDSi
  - SMART Guidelines
  - Connectathon Tracks
- Fields:
  - Name
  - Description
  - URL
  - Status
  - Owner organization
  - Contact info
  - Related states/jurisdictions
  - Visibility flag

## Topic ↔ External Initiative Mapping
- ES topic can map to many external initiatives
- External initiative topic can map to many ES topics
- Preserve external naming separately
- Preserve external status/lifecycle separately

## External Lifecycle Tracking
- Store initiative-specific lifecycle/status
- Example:
  - IIP Lifecycle
  - HL7 Ballot Stage
  - Pilot Phase
- ES stages remain independent/descriptive
